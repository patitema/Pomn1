## 2. Реализованный функционал

### 2.1 Backend (Django)

#### Модели данных (`api/models.py`)

| Модель | Поля | Описание |
|--------|------|----------|
| `User` (Django built-in) | username, email, password | Стандартная модель пользователя Django |
| `Profile` | user, phone_number | Расширенный профиль пользователя с телефоном |
| `Folder` | id, title, path, parent, user, created_at | Папки для заметок с поддержкой вложенности |
| `Note` | id, title, text, folder, user, created_at | Заметки с привязкой к папкам |
| `Link` | folder, note | Связи между заметками и папками (заготовка) |

#### API Endpoints (`api/urls.py`)

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/register/` | Регистрация нового пользователя |
| POST | `/api/login/` | Аутентификация пользователя |
| POST | `/api/logout/` | Выход из системы |
| GET | `/api/current-user/` | Получение данных текущего пользователя |
| PUT | `/api/update-profile/` | Обновление профиля |
| GET | `/api/notes/` | Список всех заметок пользователя |
| POST | `/api/notes/` | Создание заметки |
| GET | `/api/notes/<id>/` | Получение заметки по ID |
| PUT | `/api/notes/<id>/` | Обновление заметки |
| DELETE | `/api/notes/<id>/` | Удаление заметки |
| GET | `/api/folders/` | Список папок пользователя |
| POST | `/api/folders/` | Создание папки |
| GET | `/api/folders/<id>/` | Получение папки по ID |
| PUT | `/api/folders/<id>/` | Обновление папки |
| DELETE | `/api/folders/<id>/` | Удаление папки |

#### Система аутентификации
- Token-based аутентификация (Django REST Framework)
- Хранение токена в localStorage на клиенте
- Защита endpoints через `IsAuthenticated`

## 3. Детальная реализация функционала

### 3.1 Аутентификация и авторизация ✅

**Реализовано:**
- Регистрация с полями: username, email, password, phone_number
- Вход по логину и паролю
- Выход из системы с очисткой токена
- Получение данных текущего пользователя
- Обновление профиля (телефон)
- Сохранение токена в localStorage
- Protected routes (редирект на /Auth если нет токена)

**Файлы:**
- `backend/api/views.py` — функции register, login, logout, current_user, update_profile
- `frontend/src/hooks/UseUsers.js` — хук useUsers
- `frontend/src/pages/Auth/Auth.jsx` — страница входа
- `frontend/src/pages/Reg/Reg.jsx` — страница регистрации
- `frontend/src/pages/Profile/Profile.jsx` — профиль пользователя

---

### 3.2 Управление заметками ✅

**Реализовано:**
- CRUD операции с заметками
- Привязка заметок к папкам
- Фильтрация заметок по пользователю
- Отображение даты создания
- Поиск по заметкам

**Не реализовано:**
- Markdown-редактор
- Теги для заметок
- Дедлайны для задач

**Файлы:**
- `backend/api/models.py` — модель Note
- `backend/api/views.py` — notes_list, note_detail
- `frontend/src/pages/Notes/Notes.jsx` — страница с графом
- `frontend/src/pages/Folder-view/Folder.jsx` — управление заметками

---

### 3.3 Управление папками ✅

**Реализовано:**
- CRUD операции с папками
- Вложенные папки (иерархия)
- Автоматическое формирование path
- Drag-and-drop для перемещения заметок между папками
- Поиск по папкам

**Файлы:**
- `backend/api/models.py` — модель Folder

### 3.4 Визуализация графа знаний ⚠️

**Реализовано:**

- Силавая модель графа (force simulation)
- Перетаскивание узлов графа
- Цветовая группировка узлов

- Модель Link не используется

**Файлы:**
- `frontend/src/components/forceGraphRender/forceGraphRender.jsx`
- `backend/api/models.py` — модель Link (заготовка)

## 6. Известные проблемы

### 6.1 Backend
- Модель `Link` создана, но не используется
- Отсутствует endpoint для графа (`/graph.json`)
- Нет автоматического построения связей между заметками

### 6.2 Frontend
- Граф загружает данные из `/graph.json`, который не существует
- Страница `/Tasks` пустая
- Отсутствует Markdown-редактор
- Нет валидации форм на frontend

### 6.3 Конфигурация
- База данных MySQL требует внешний сервер (127.127.126.5)
- DEBUG = True (не для production)
- SECRET_KEY захардкожен в settings.py
