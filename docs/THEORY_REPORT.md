---

## 1. Общая информация

### 1.1 Тема проекта
**Сайт для отслеживания и запоминания заметок «Pomni»**

### 1.2 Назначение системы
АИС «Pomni» — это веб-сервис, предназначенный для:
- Хранения персональной информации
- Структурирования заметок и задач
- Анализа и визуализации связей между данными

### 1.3 Целевая аудитория
- Студенты
- Специалисты
- Фрилансеры
- Все, кто работает с большими объёмами информации

---

## 2. Функциональные требования

### 2.1 Основные модули системы

#### А1. Модуль аутентификации и авторизации
- Регистрация пользователя
- Вход в систему (логин/пароль)
- Выход из системы
- Управление профилем пользователя
- Проверка данных и создание защищённой сессии

#### А2. Модуль управления заметками
- Создание заметок
- Редактирование заметок
- Удаление заметок
- Поддержка Markdown-редактора
- Привязка тегов к заметкам
- Установка дедлайнов для задач
- Организация заметок по папкам (иерархическая структура)

#### А3. Модуль управления папками
- Создание папок
- Вложенные папки (подпапки)
- Перемещение заметок между папками
- Удаление папок

#### А4. Модуль визуализации графа знаний
- Интерактивный граф связей между заметками
- Визуализация ассоциативных связей
- Отображение структуры знаний пользователя
- Анализ массива данных и генерация визуальной структуры

#### А5. Модуль отчётности и экспорта
- Формирование отчётности по заметкам
- Экспорт данных (архивы, файлы)
- Создание экспортных файлов для пользователя

#### А6. Модуль управления задачами (таск-трекинг)
- Отслеживание статусов выполнения задач
- Привязка задач к заметкам
- Управление дедлайнами

---

## 3. Нефункциональные требования

### 3.1 Требования к безопасности
- Защита персональных данных пользователей
- Соответствие законам РФ о защите ПД
- Регламенты безопасности данных
- Сессионные токены для аутентификации

### 3.2 Требования к доступности
- Круглосуточная доступность платформы
- Доступ с любого устройства
- Веб-интерфейс (не требует установки)

### 3.3 Требования к масштабируемости
- Архитектура должна обеспечивать масштабируемость
- Безопасность данных при росте нагрузки

## 5. Сравнительный анализ с аналогами

### Таблица 1. Сравнение с конкурентами

| Критерии | Obsidian | Notion | Google Keep | Pomni (требуется) |
|----------|----------|--------|-------------|-------------------|
| Визуальный граф связей | + | – | – | **+** |
| Таск-трекинг (статусы) | – | + | + | **+** |
| Markdown-редактор | + | + | – | **+** |
| Облачная синхронизация | +/– | + | + | **+** |
| Личный кабинет и безопасность | + | + | + | **+** |

### 5.1 Уникальные преимущества Pomni
1. **Простота использования** — в отличие от Obsidian, не требует сложной настройки
2. **Визуализация связей** — в отличие от Notion, есть граф знаний
3. **Облачная синхронизация** — доступ с любого устройства
4. **Единая платформа** — объединение заметок и задач в одном месте

## 9. Выводы

Система «Pomni» должна объединить лучшие качества существующих решений:
1. **Визуальный граф связей** как в Obsidian
2. **Таск-трекинг** как в Notion
3. **Простоту использования** как в Google Keep
4. **Облачную синхронизацию** для доступа с любого устройства
5. **Безопасность данных** с соблюдением законодательства РФ

Ключевая особенность — создание «второго мозга» для пользователя, который снижает когнитивную нагрузку и помогает систематизировать личную информацию.
